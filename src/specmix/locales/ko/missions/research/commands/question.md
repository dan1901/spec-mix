## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

전체 연구 과정을 안내할 명확하고 집중된 연구 질문을 정의합니다. 이것이 연구 프로젝트의 기초입니다.

## 실행 흐름

### 1. 연구 디렉토리 생성

디렉토리 구조 생성:
```
specs/[###-연구-주제]/
├── research-question.md    # 이 출력물
├── methodology.md          # /spec-mix.methodology로 생성
├── literature-review.md    # /spec-mix.literature로 생성
├── data-sources.md         # /spec-mix.search로 생성
├── evidence/               # /spec-mix.collect로 생성
├── findings.md             # /spec-mix.findings로 생성
├── analysis.md             # /spec-mix.synthesize로 생성
└── bibliography.md         # 전체 과정에서 관리
```

### 2. 사용자 입력 분석

사용자의 연구 주제에서 추출:
- **핵심 주제**: 어떤 분야인가?
- **관점**: 어떤 특정 측면인가?
- **목적**: 왜 중요한가?
- **제약조건**: 시간, 자원, 접근 제한

### 3. 연구 질문 수립

**주요 질문 요건**:
- 구체적 (너무 광범위하지 않음)
- 답변 가능 (가용 자원으로)
- 관련성 (의미있는 영향)
- 객관적 (미리 정해진 답으로 유도하지 않음)

**질문 유형**:
| 유형 | 템플릿 | 예시 |
|------|--------|------|
| 탐색적 | "...의 핵심 요소는 무엇인가?" | "AI 도입의 핵심 요소는 무엇인가?" |
| 서술적 | "X는 어떻게 작동하는가?" | "트랜스포머 아키텍처는 어떻게 작동하는가?" |
| 비교 | "X와 Y는 어떻게 비교되는가?" | "RAG와 파인튜닝은 어떻게 비교되는가?" |
| 인과 | "X가 Y에 미치는 영향은?" | "컨텍스트 길이가 정확도에 미치는 영향은?" |
| 평가 | "Y를 위해 X는 얼마나 효과적인가?" | "코드 생성을 위해 프롬프트 엔지니어링은 얼마나 효과적인가?" |

### 4. 보조 질문 정의

주요 질문을 3-5개의 보조 질문으로 분해:
- 각각 독립적으로 연구 가능해야 함
- 함께 모여 주요 질문을 포괄적으로 다뤄야 함
- 논리적 순서로 정렬 (기초 → 심화)

### 5. 범위 설정

**포함 범위** - 명시적으로 포함:
- 조사할 특정 측면
- 기간 (관련 시)
- 지리적/도메인 경계
- 고려할 출처 유형

**제외 범위** - 명시적으로 제외:
- 관련되었지만 부차적인 주제
- 가용 자원이 필요한 영역
- 향후 연구를 위한 질문

### 6. 성공 기준 정의

연구는 다음 조건 충족 시 성공:
- [ ] 주요 질문이 5개 이상의 신뢰할 수 있는 출처의 증거로 답변됨
- [ ] 모든 보조 질문이 다뤄짐
- [ ] 발견사항이 적절한 인용과 함께 문서화됨
- [ ] 한계와 불확실성이 인정됨
- [ ] 결론이 가정이 아닌 증거로 뒷받침됨

### 7. 제약조건 식별

문서화:
- **시간**: 가용 연구 기간
- **자원**: 사용 가능한 도구, 데이터베이스, 접근권
- **전문성**: 도메인 지식 제한
- **접근성**: 유료화, 언어 장벽 등

### 8. research-question.md 작성

`.spec-mix/active-mission/templates/research-question-template.md`의 템플릿 사용

위 분석의 구체적 내용으로 모든 섹션 작성.

### 9. 참고문헌 초기화

빈 `bibliography.md` 생성:
```markdown
# 참고문헌: [주제]

## 인용 형식
[지정: APA/MLA/Chicago/IEEE]

## 출처

### 주요 출처
| ID | 인용 | 유형 | 접근일 | 품질 | 사용처 |
|----|------|------|--------|------|--------|

### 보조 출처
| ID | 인용 | 유형 | 접근일 | 품질 | 사용처 |
|----|------|------|--------|------|--------|

## 참고
- 품질 등급: A (동료 검토), B (신뢰할 수 있는 출처), C (비공식/제한적)
```

## 출력

보고:
- 생성된 디렉토리 경로
- 연구 질문 요약
- 보조 질문 수
- 정의된 범위 경계
- 다음 단계: `/spec-mix.methodology` 또는 `/spec-mix.search`

## 품질 체크리스트

완료 전 확인:
- [ ] 질문이 구체적이고 답변 가능한가
- [ ] 질문이 유도하거나 편향되지 않았는가
- [ ] 범위가 명확히 한정되었는가
- [ ] 성공 기준이 측정 가능한가
- [ ] 제약조건이 현실적인가

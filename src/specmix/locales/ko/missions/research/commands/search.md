## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

연구 질문에 답하기 위한 관련 출처를 체계적으로 검색하고 수집합니다. AI 웹 검색 기능을 활용하여 신뢰할 수 있는 정보를 찾습니다.

## 전제조건

- `research-question.md`가 존재해야 함
- `methodology.md`가 존재해야 함 (권장)

## 실행 흐름

### 1. 컨텍스트 로드

읽기:
- `research-question.md` - 답변할 질문
- `methodology.md` - 검색 전략 (존재 시)
- `bibliography.md` - 이미 수집된 출처

### 2. 검색 쿼리 준비

각 보조 질문별 준비:

**쿼리 수립**:
```
주요: "[핵심 개념] [특정 측면]"
변형:
- 동의어: "[대안 용어] [측면]"
- 확장: "[일반 주제] 개요"
- 축소: "[특정 하위주제] [상세]"
```

**검색 연산자**:
- 정확한 구문에 따옴표 사용
- 특정 출처에 site:domain 추가
- 논문에 filetype:pdf 추가
- 최신성을 위해 연도 범위 추가

### 3. 웹 검색 실행

**중요**: 각 쿼리에 WebSearch 도구를 사용합니다.

**검색 순서**:
1. 주요 쿼리로 시작
2. 결과 검토, 부족한 부분 기록
3. 변형 쿼리 실행
4. 특정 도메인 검색:
   - `site:arxiv.org` - 학술 논문
   - `site:github.com` - 코드/구현
   - `site:medium.com OR site:dev.to` - 기술 블로그
   - `site:stackoverflow.com` - 커뮤니티 지식

**각 검색별**:
- WebSearch로 쿼리 실행
- 상위 10-15개 결과 검토
- 관련성과 품질 평가
- 유망한 출처 기록

### 4. 출처 평가

**빠른 평가 체크리스트**:
- [ ] 연구 질문과 관련 있는가?
- [ ] 저자/기관이 신뢰할 수 있는가?
- [ ] 날짜가 허용 범위인가 (방법론 기준)?
- [ ] 증거 기반인가 (단순 의견이 아닌)?
- [ ] 접근 가능한가 (유료벽 없이)?

**품질 등급**:
- **A**: 동료 검토, 권위 있는 출처
- **B**: 신뢰할 수 있는 기관, 잘 인용됨
- **C**: 비공식이지만 유용함

### 5. 출처 기록

각 가치있는 출처를 `data-sources.md`에 추가:

```markdown
## 출처: [제목]

**URL**: [링크]
**유형**: [논문/기사/문서/블로그/보고서]
**저자**: [이름/기관]
**날짜**: [출판일]
**접근일**: [오늘 날짜]
**품질**: [A/B/C]

**관련성**:
- 다루는 질문: [Q1, Q2 등]
- 핵심 주제: [다루는 주제]

**요약**:
[내용에 대한 2-3문장 요약]

**주요 인용** (나중 인용용):
> "[중요한 인용 1]" (섹션/페이지)
> "[중요한 인용 2]" (섹션/페이지)

**참고**:
[관찰사항, 한계, 편향 등]
```

### 6. 참고문헌 업데이트

각 출처를 `bibliography.md`에 추가:

```markdown
| ID | 인용 | 유형 | 접근일 | 품질 | 사용처 |
|----|------|------|--------|------|--------|
| S01 | 저자 (연도). 제목. 출처. URL | 논문 | 2025-01-07 | A | Q1, Q3 |
```

### 7. 커버리지 추적

커버리지 매트릭스 생성:

```markdown
## 검색 커버리지

| 질문 | 발견된 출처 | A등급 | B등급 | 부족분 |
|------|-------------|-------|-------|--------|
| Q1 | 5 | 2 | 3 | 없음 |
| Q2 | 3 | 1 | 1 | 더 많은 주요 출처 필요 |
| Q3 | 2 | 0 | 2 | A등급 출처 없음 |
```

### 8. 필요시 반복

**부족분이 있으면**:
- 검색 쿼리 개선
- 대안 용어 시도
- 다른 출처 카테고리 검색
- 범위를 약간 확장 고려

**최소 요건**:
- 보조 질문당 최소 3개 출처
- 주요 주제당 최소 1개 A등급 출처
- C등급 출처만 있는 질문 없음

## 출력

보고:
- 수집된 총 출처 수
- 질문별 커버리지
- 품질 분포 (A/B/C)
- 식별된 부족분
- 다음 단계: `/spec-mix.literature` 또는 `/spec-mix.collect`

## 검색 팁

**학술 출처**:
- "systematic review" + 주제
- "meta-analysis" + 주제
- "[주제] survey paper"

**기술 출처**:
- "[기술] documentation"
- "[도구] best practices"
- "[프레임워크] architecture"

**최근 동향**:
- "[주제] 2024" 또는 "[주제] 2025"
- "[주제] latest research"
- "[주제] state of the art"

## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

각 연구 질문에 답하기 위한 구체적 증거를 출처에서 수집합니다. 쉬운 검색과 인용을 위해 증거를 정리합니다.

## 전제조건

- `research-question.md`가 존재해야 함
- `data-sources.md`에 출처가 나열되어 있어야 함
- `literature-review.md`가 완성되어 있어야 함 (권장)

## 실행 흐름

### 1. 컨텍스트 로드

읽기:
- `research-question.md` - 답변할 질문
- `methodology.md` - 증거 품질 기준
- `data-sources.md` - 추출할 출처
- `literature-review.md` - 식별된 주제

### 2. 증거 디렉토리 생성

```
FEATURE_DIR/evidence/
├── Q1-[질문-슬러그]/
│   ├── evidence-01.md
│   ├── evidence-02.md
│   └── summary.md
├── Q2-[질문-슬러그]/
│   └── ...
└── cross-cutting/
    └── ...
```

### 3. 질문별 증거 추출

각 보조 질문에 대해 체계적으로 증거 추출:

**증거 추출 템플릿**:
```markdown
# 증거: [간단한 설명]

## 메타데이터
- **출처**: [참고문헌의 출처 ID]
- **질문**: Q[#]
- **유형**: [통계/인용/발견사항/예시/데이터]
- **품질**: [A/B/C - 출처에서 상속]

## 증거 내용

### 원시 증거
> "[정확한 인용문 또는 데이터 포인트]"
> — 출처: [인용]

### 맥락
[이 증거의 맥락은 무엇인가? 출처가 어떤 질문에 답하고 있었는가?]

### 해석
[이 증거가 우리 연구 질문에 대해 무엇을 시사하는가?]

## 신뢰성 평가

- **직접성**: [우리 질문에 대한 직접/간접 증거]
- **뒷받침**: [이를 지지하는 다른 출처? ID 나열]
- **한계**: [이 증거에 대한 주의사항]

## 태그
[키워드1], [키워드2], [질문-번호]
```

### 4. 수집할 증거 유형

**정량적 증거**:
- 통계와 메트릭
- 설문 결과
- 성능 벤치마크
- 비교 데이터

**정성적 증거**:
- 전문가 의견
- 사례 연구 발견사항
- 이론적 논거
- 관찰된 패턴

**혼합 증거**:
- 둘 다 포함하는 연구 결과
- 산업 보고서
- 체계적 검토

### 5. 품질 관리

각 증거에 대해 확인:

| 확인 | 통과 | 참고 |
|------|------|------|
| 출처가 참고문헌에 있음 | [ ] | |
| 인용이 정확함 | [ ] | |
| 맥락이 보존됨 | [ ] | |
| 체리피킹 아님 | [ ] | |
| 한계가 기록됨 | [ ] | |

### 6. 증거 커버리지 추적

`evidence/coverage.md` 생성:

```markdown
# 증거 커버리지 매트릭스

## 질문별

| 질문 | 증거 수 | A등급 | B등급 | 충분함? |
|------|---------|-------|-------|---------|
| Q1 | 8 | 3 | 4 | 예 |
| Q2 | 4 | 1 | 2 | 더 필요 |
| Q3 | 6 | 2 | 3 | 예 |

## 주제별

| 주제 | 증거 수 | 출처 | 참고 |
|------|---------|------|------|
| [주제 1] | 5 | S01, S03, S07 | 강한 커버리지 |
| [주제 2] | 3 | S02, S04 | 뒷받침 필요 |

## 식별된 격차
- [격차 1]: [측면]에 대한 직접 증거 없음
- [격차 2]: [주장]에 단일 출처만 있음

## 상충 증거
- [주제]: E-01 vs E-05 (충돌하는 발견사항)
```

### 7. 질문 요약 생성

각 질문 디렉토리에 `summary.md` 생성:

```markdown
# 증거 요약: Q[#]

## 질문
[전체 보조 질문]

## 증거 개요
- 총 건수: [#]
- 품질 분포: [# A, # B, # C]
- 사용된 출처: [목록]

## 핵심 증거

### [답변 방향] 지지
1. **E-01**: [간단 요약] (품질: A)
2. **E-03**: [간단 요약] (품질: B)

### 반박/복잡화
1. **E-05**: [간단 요약] (품질: B)

## 예비 답변
수집된 증거를 바탕으로, [질문에 대한 잠정적 답변]

## 신뢰도 수준
[높음/중간/낮음] - [이유] 때문

## 남은 격차
[어떤 증거가 여전히 필요한가?]
```

### 8. 상호참조 확인

확인:
- 모든 증거가 참고문헌으로 추적됨
- 고아 증거 없음 (인용되지 않은 출처)
- 지지되지 않은 주장 없음 (각 주장에 증거)

## 출력

보고:
- 수집된 총 증거 건수
- 질문별 커버리지
- 품질 분포
- 식별된 격차
- 발견된 모순
- 다음 단계: `/spec-mix.synthesize` 또는 `/spec-mix.findings`

## 증거 수집 원칙

- **완전성**: 지지와 반박 증거 모두 수집
- **정확성**: 정확히 인용, 부정확하게 의역하지 않음
- **맥락**: 증거의 원래 맥락 기록
- **추적성**: 모든 건이 참고문헌에 연결
- **정직성**: 불편한 증거 무시하지 않음

---

## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

연구 질문, 범위 또는 방법론의 모호성을 탐지하고 줄입니다. 데이터 수집 전에 연구가 집중되고 달성 가능한지 확인합니다.

## 전제조건

- `research-question.md`가 존재해야 함

## 실행 흐름

### 1. 연구 질문 로드

읽기:
- `research-question.md` - 주요 및 보조 질문
- `methodology.md` - 존재 시, 정렬 확인

### 2. 모호성 스캔

다음 카테고리에 대해 연구 질문 분석:

**질문 명확성**:
- 주요 질문이 충분히 구체적인가?
- 보조 질문이 구별되고 중복되지 않는가?
- 질문에 숨겨진 가정이 있는가?

**범위 정의**:
- 포함 범위 경계가 명확한가?
- 제외 범위 항목이 명시적으로 기술되었는가?
- 가용 자원으로 범위가 달성 가능한가?

**성공 기준**:
- 성공 기준이 측정 가능한가?
- 성공을 객관적으로 검증할 수 있는가?
- 명시되지 않은 기대가 있는가?

**방법론 정렬** (methodology.md가 존재하면):
- 접근법이 질문 유형과 일치하는가?
- 데이터 출처가 적절한가?
- 분석 프레임워크가 적합한가?

**제약조건 & 실현 가능성**:
- 시간 제약이 현실적인가?
- 자원 제한이 인정되었는가?
- 접근 제한이 고려되었는가?

**용어**:
- 핵심 용어가 정의되었는가?
- 모호한 문구가 있는가?
- 도메인 전문용어가 설명되었는가?

각 카테고리에 대해 상태 표시: **명확 / 부분적 / 누락**

### 3. 명확화 질문 생성

우선순위가 지정된 질문 생성 (최대 5개):

**질문 기준**:
- 연구 방향, 범위 또는 품질에 실질적인 영향을 미쳐야 함
- 짧은 답변이나 객관식으로 답변 가능해야 함
- 복잡성을 추가하지 않고 모호성을 줄여야 함

**우선순위 순서**:
1. 주요 연구 질문에 영향을 미치는 질문
2. 범위 경계에 영향을 미치는 질문
3. 방법론에 영향을 미치는 질문
4. 성공 기준에 영향을 미치는 질문
5. 용어에 영향을 미치는 질문

### 4. 대화형 명확화

한 번에 하나의 질문 제시:

**객관식 질문용**:
```markdown
**질문 [#]**: [질문 텍스트]

**권장**: 옵션 [X] - [연구 품질에 왜 이것이 최선인지 이유]

| 옵션 | 설명 |
|------|------|
| A | [옵션 A] |
| B | [옵션 B] |
| C | [옵션 C] |

옵션 문자, "권장", 또는 직접 답변 (5단어 이하)으로 응답하세요.
```

**단답형 질문용**:
```markdown
**질문 [#]**: [질문 텍스트]

**제안**: [제안 답변] - [간략한 이유]

"제안"으로 수락하거나, 직접 답변 (5단어 이하)을 제공하세요.
```

**각 답변 후**:
1. 명확화 기록
2. `research-question.md` 즉시 업데이트
3. 다음 질문으로 이동

**종료 시점**:
- 모든 중요한 모호성 해결됨
- 사용자가 완료 신호 ("완료", "진행")
- 5개 질문 완료

### 5. 통합

각 답변 후 `research-question.md` 업데이트:

**명확화 섹션 추가** (존재하지 않으면):
```markdown
## 명확화

### 세션 YYYY-MM-DD
- Q: [질문] → A: [답변]
```

**관련 섹션에 적용**:
- 질문 명확성 → 주요/보조 질문 업데이트
- 범위 변경 → 포함 범위 / 제외 범위 업데이트
- 성공 기준 → 성공 기준 섹션 업데이트
- 용어 → 용어집에 추가 또는 맥락에서 명확화
- 제약조건 → 제약조건 섹션 업데이트

### 6. 검증

각 업데이트 후 확인:
- [ ] 모순되는 진술이 남아있지 않음
- [ ] 명확화가 적절한 섹션에 통합됨
- [ ] 연구 질문이 일관성 유지
- [ ] 범위가 여전히 달성 가능

### 7. 완료 보고

```markdown
## 명확화 요약

**질문된 질문 수**: [#]
**업데이트된 섹션**: [목록]

### 커버리지 상태
| 카테고리 | 상태 | 참고 |
|----------|------|------|
| 질문 명확성 | 명확/부분적/누락 | |
| 범위 정의 | 명확/부분적/누락 | |
| 성공 기준 | 명확/부분적/누락 | |
| 방법론 | 명확/부분적/누락 | |
| 제약조건 | 명확/부분적/누락 | |
| 용어 | 명확/부분적/누락 | |

### 미해결 항목
[남은 모호성]

### 권장 다음 단계
[/spec-mix.methodology 또는 /spec-mix.search]
```

## 행동 규칙

- 의미있는 모호성이 없으면: "연구 질문이 잘 정의되어 있습니다. 방법론 또는 검색을 진행할 준비가 되었습니다."
- `research-question.md`가 없으면: 먼저 `/spec-mix.question` 실행 안내
- 5개 질문 초과 금지
- 사용자 조기 종료 존중 ("완료", "진행", "중지")
- 완벽주의가 아닌 연구 품질에 집중

## 명확화 질문 예시

**범위**:
- "이 연구에 역사적 분석 (2020년 이전)을 포함해야 하나요, 아니면 현재 상태에만 집중해야 하나요?"
- "이 연구는 학술 출처에만 제한되나요, 아니면 산업 보고서도 포함해야 하나요?"

**질문 정제**:
- "'[모호한 용어]'는 X 또는 Y를 의미할 수 있습니다. 어떤 해석을 사용해야 하나요?"
- "[주제]를 다른 [차원]에 걸쳐 비교해야 하나요, 아니면 단일 [차원]에 집중해야 하나요?"

**방법론**:
- "질문 유형을 고려할 때, 정량적 데이터를 우선시해야 하나요 아니면 정성적 분석을 우선시해야 하나요?"
- "결론에 허용되는 신뢰도 수준은 무엇인가요: 높은 신뢰도만 또는 중간 신뢰도 발견사항도 포함?"

**실현 가능성**:
- "시간 제약을 고려할 때, 출처를 [#]개로 제한해야 하나요 아니면 일부 영역의 부분적 커버리지를 수용해야 하나요?"

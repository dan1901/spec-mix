## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

연구의 품질을 보장하고, 편향을 식별하며, 결론이 증거에 의해 적절히 뒷받침되는지 확인하기 위한 체계적 검증을 수행합니다.

## 전제조건

- `findings.md`가 완성되어 있어야 함
- `analysis.md`가 존재해야 함
- `bibliography.md`가 완성되어 있어야 함

## 실행 흐름

### 1. 모든 산출물 로드

읽기:
- `research-question.md`
- `methodology.md`
- `literature-review.md`
- `analysis.md`
- `findings.md`
- `bibliography.md`
- 모든 `evidence/Q#/summary.md` 파일

### 2. 증거 추적성 확인

findings.md의 모든 주장이 증거로 추적되는지 확인:

```markdown
## 증거 추적성 감사

| 발견사항 | 주장 | 증거 ID | 출처 | 확인됨 |
|----------|------|---------|------|--------|
| F1 | [주장 텍스트] | E-01, E-03 | S01, S04 | [예/아니오] |
| F1 | [주장 텍스트] | E-05 | S07 | [예/아니오] |
| F2 | [주장 텍스트] | E-02 | S02 | [예/아니오] |

### 발견된 지지되지 않은 주장
- [증거 없는 주장 목록]

### 사용되지 않은 증거
- [참조되지 않은 수집된 증거 목록]
```

### 3. 출처 품질 감사

참고문헌 품질 검토:

```markdown
## 출처 품질 감사

### 품질 분포
| 등급 | 수 | 비율 | 임계값 | 상태 |
|------|-----|------|--------|------|
| A | [#] | [%] | ≥30% | [통과/실패] |
| B | [#] | [%] | ≥40% | [통과/실패] |
| C | [#] | [%] | ≤30% | [통과/실패] |

### 중요 발견사항 확인
각 높은 신뢰도 발견사항에 대해, A등급 출처 지지 확인:

| 발견사항 | 신뢰도 | A등급 출처 | 상태 |
|----------|--------|------------|------|
| F1 | 높음 | S01, S05 | 통과 |
| F2 | 높음 | 없음 | 실패 - 하향 조정 |
```

### 4. 편향 탐지

일반적인 연구 편향 확인:

```markdown
## 편향 평가

### 확증 편향
- [ ] 반박 증거가 적극적으로 탐색됨
- [ ] 반박 증거가 발견사항에서 인정됨
- [ ] 대안적 해석이 고려됨

**발견된 문제**: [없음 / 문제 목록]

### 선택 편향
- [ ] 출처 선택 기준이 문서화됨
- [ ] 출처가 다양한 관점을 대표함
- [ ] 관점의 체계적 배제가 없음

**발견된 문제**: [없음 / 문제 목록]

### 최신성 편향
- [ ] 기초/오래된 출처가 포함됨
- [ ] 최신 출처에만 과도하게 의존하지 않음

**발견된 문제**: [없음 / 문제 목록]

### 권위 편향
- [ ] 주장이 출처 명성만이 아닌 증거로 평가됨
- [ ] 덜 유명한 출처의 반박도 고려됨

**발견된 문제**: [없음 / 문제 목록]

### 가용성 편향
- [ ] 명백하지 않은 출처를 찾으려는 노력이 있었음
- [ ] 유료벽/제한된 출처가 관련 시 인정됨

**발견된 문제**: [없음 / 문제 목록]
```

### 5. 논리 검증

추론 품질 확인:

```markdown
## 논리 검증

### 결론 지지 확인
각 결론에 대해 논리적 연결 확인:

| 결론 | 지지 증거 | 논리 유효 | 문제 |
|------|-----------|-----------|------|
| [C1] | E-01, E-03, E-05 | 예 | 없음 |
| [C2] | E-02 | 약함 | 단일 출처 |

### 논리적 오류 확인
- [ ] 거짓 인과관계 없음 (상관관계 ≠ 인과관계)
- [ ] 성급한 일반화 없음 (작은 표본 → 넓은 주장)
- [ ] 증거 없는 권위에 호소 없음
- [ ] 허수아비 논법 없음 (출처 오역)
- [ ] 체리피킹 없음 (선택적 증거 사용)

**발견된 오류**: [없음 / 위치와 함께 목록]
```

### 6. 완전성 확인

연구 질문 커버리지 확인:

```markdown
## 완전성 확인

### 연구 질문 커버리지
| 질문 | 다뤄짐 | 증거 | 신뢰도 명시 | 상태 |
|------|--------|------|-------------|------|
| 주요 | 예/아니오 | [ID들] | 예/아니오 | [통과/실패] |
| Q1 | 예/아니오 | [ID들] | 예/아니오 | [통과/실패] |
| Q2 | 예/아니오 | [ID들] | 예/아니오 | [통과/실패] |

### 누락된 요소
- [ ] 모든 질문이 다뤄짐
- [ ] 모든 방법론 단계가 완료됨
- [ ] 모든 한계가 문서화됨
- [ ] 모든 모순이 다뤄짐
```

### 7. 신뢰도 보정

신뢰도 수준 검토:

```markdown
## 신뢰도 보정

| 발견사항 | 명시된 신뢰도 | 증거 품질 | 출처 수 | 권장 |
|----------|--------------|----------|---------|------|
| F1 | 높음 | A: 2, B: 1 | 3 | 높음 (적절) |
| F2 | 높음 | A: 0, B: 2 | 2 | 중간 (하향) |
| F3 | 중간 | A: 1, B: 2 | 3 | 중간 (적절) |

### 필요한 조정
- [변경되어야 할 신뢰도 수준 목록]
```

### 8. 검증 보고서 생성

`FEATURE_DIR/validation-report.md` 생성:

```markdown
# 검증 보고서: [주제]

## 전체 평가
**상태**: [통과 / 문제와 함께 통과 / 수정 필요]

## 요약 메트릭
- 증거 추적성: [%] 주장 확인됨
- 출처 품질: [%] A+B 등급
- 편향 확인: [#]/[#] 통과
- 논리 검증: [#]/[#] 통과
- 완전성: [#]/[#] 질문 다뤄짐

## 발견된 문제

### 심각 (반드시 수정)
1. [문제 설명]
   - 위치: [발견사항 어디에]
   - 수정: [권장 조치]

### 중요 (수정 권장)
1. [문제 설명]

### 사소 (수정 고려)
1. [문제 설명]

## 신뢰도 조정
[권장 변경사항 목록]

## 권고사항
1. [필요한 조치]
2. [필요한 조치]

## 검증 체크리스트
- [ ] 모든 주장이 증거로 추적 가능
- [ ] 출처 품질이 임계값 충족
- [ ] 주요 편향 미탐지
- [ ] 논리가 건전함
- [ ] 모든 질문이 다뤄짐
- [ ] 신뢰도 수준이 적절함
- [ ] 한계가 문서화됨

## 검증자 참고
[추가 관찰사항]
```

## 출력

보고:
- 전체 검증 상태
- 심각도별 발견된 문제
- 필요한 신뢰도 조정
- 최종화 전 필요한 조치
- 다음 단계: 문제 해결 또는 연구 최종화

## 검증 기준

**통과**: 모든 중요 확인 통과, 사소한 문제만
**문제와 함께 통과**: 심각한 문제 없음, 일부 중요 수정 필요
**수정 필요**: 심각한 문제 발견, 최종화 불가

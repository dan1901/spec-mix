## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

연구 질문에 답하기 위한 체계적 접근법을 수립합니다. 어떤 데이터를 어디서 어떻게 수집하고 분석할지 정의합니다.

## 전제조건

- `research-question.md`가 존재하고 완성되어 있어야 함

## 실행 흐름

### 1. 컨텍스트 로드

읽기:
- `research-question.md` - 주요 및 보조 질문
- `specs/constitution.md` - 연구 원칙 (존재 시)

### 2. 연구 접근법 선택

질문 유형에 따라 선택:

| 질문 유형 | 권장 접근법 | 데이터 유형 |
|-----------|-------------|-------------|
| 탐색적 | 문헌 검토 + 전문가 출처 | 정성적 |
| 서술적 | 문서 분석 + 사례 연구 | 혼합 |
| 비교 | 체계적 비교 프레임워크 | 정량적 + 정성적 |
| 인과 | 증거 삼각검증 | 다중 출처 |
| 평가 | 기준 기반 평가 | 메트릭 + 정성적 |

### 3. 데이터 수집 전략 정의

**출처 카테고리**:

1. **학술 출처**
   - 동료 검토 논문 (arXiv, 저널)
   - 학회 논문집
   - 학위 논문

2. **기술 출처**
   - 공식 문서
   - 기술 블로그 (신뢰할 수 있는)
   - GitHub 저장소
   - Stack Overflow (패턴용)

3. **산업 출처**
   - 산업 보고서
   - 사례 연구
   - 백서
   - 뉴스 기사 (신뢰할 수 있는 매체)

4. **전문가 출처**
   - 인터뷰/팟캐스트
   - 컨퍼런스 발표
   - 전문가 블로그

**각 보조 질문별** 지정:
- 가장 관련성 높은 출처 카테고리
- 필요한 최소 출처 수
- 품질 임계값 (A/B/C 등급)

### 4. 검색 전략 수립

각 보조 질문별 정의:
```markdown
## Q1: [보조 질문]

**키워드**: [주요], [부차], [대안]
**불리언 쿼리**: "용어1" AND ("용어2" OR "용어3") NOT "제외"
**출처**: [학술, 기술, 산업]
**기간**: [관련 시]
**언어**: [ko/en 또는 지정]
**최소 출처 수**: [숫자]
```

### 5. 분석 프레임워크 정의

**정성적 데이터용**:
- 주제 분석 (패턴 식별)
- 내용 분석 (정보 분류)
- 비교 분석 (출처 대조)

**정량적 데이터용**:
- 통계적 비교 (해당 시)
- 추세 분석
- 벤치마크 비교

**증거 삼각검증**:
- 핵심 주장에 3개 이상의 독립 출처 필요
- 상충되는 증거 문서화
- 합의 vs. 논쟁 영역 기록

### 6. 품질 기준 설정

**출처 품질 평가**:
| 기준 | A (높음) | B (중간) | C (낮음) |
|------|----------|----------|----------|
| 권위 | 동료 검토, 인정받는 전문가 | 알려진 기관 | 미상/비공식 |
| 최신성 | < 2년 | 2-5년 | > 5년 |
| 관련성 | 질문에 직접 답변 | 관련 주제 | 부수적 |
| 객관성 | 균형잡힌, 인용됨 | 약간의 편향 | 홍보성/편향됨 |

**최소 품질 요건**:
- 출처의 최소 50%가 A 또는 B 등급이어야 함
- C 등급 출처만으로 결론 도출 금지
- 상충되는 주장은 A 등급 출처로 해결 필요

### 7. 산출물 정의

| 단계 | 산출물 | 설명 |
|------|--------|------|
| 검색 | `data-sources.md` | 큐레이션된 출처 목록 |
| 문헌 | `literature-review.md` | 기존 지식 요약 |
| 수집 | `evidence/*.md` | 정리된 증거 파일 |
| 종합 | `findings.md` | 인용이 포함된 핵심 발견사항 |
| 분석 | `analysis.md` | 교차 분석 |

### 8. methodology.md 작성

`FEATURE_DIR/methodology.md` 생성:

```markdown
# 연구 방법론: [주제]

## 연구 접근법
[선택한 접근법과 근거]

## 데이터 수집 전략

### 출처 카테고리
[어떤 카테고리와 그 이유]

### 질문별 검색 전략
[상세 검색 계획]

## 분석 프레임워크
[데이터 분석 방법]

## 품질 기준
[최소 기준]

## 산출물
[생산할 것]

## 한계
[방법론의 알려진 제약]
```

보고:

- 대상 출처 카테고리 수
- 정의된 검색 전략
- 수립된 품질 기준
- 다음 단계: `/spec-mix.search` 또는 `/spec-mix.literature`

---

## 사용자 입력

```text
$ARGUMENTS
```

(입력이 비어있지 않다면) 진행하기 전에 반드시 사용자 입력을 고려해야 합니다.

## 목적

연구의 어느 단계에서든 품질 체크리스트를 생성하고 평가하여 기준 충족을 보장합니다. 연구 과정 전반에 걸쳐 여러 번 실행 가능합니다.

## 사용 시기

- 연구 질문 정의 후
- 문헌 검토 완료 후
- 증거 수집 후
- 발견사항 최종화 전
- 최종 완료 전 확인으로

## 실행 흐름

### 1. 연구 단계 판단

존재하는 산출물 확인:
- `research-question.md` → 질문 단계 완료
- `methodology.md` → 방법론 단계 완료
- `data-sources.md` → 검색 단계 완료
- `literature-review.md` → 문헌 단계 완료
- `evidence/` → 수집 단계 완료
- `analysis.md` → 분석 단계 완료
- `findings.md` → 문서화 단계 완료

### 2. 단계별 체크리스트 생성

`FEATURE_DIR/checklists/quality-[단계].md` 생성:

### 질문 단계용:

```markdown
# 품질 체크리스트: 연구 질문

## 질문 명확성
- [ ] 주요 질문이 구체적이고 집중됨
- [ ] 주요 질문이 가용 자원으로 답변 가능함
- [ ] 질문이 유도하거나 편향되지 않음
- [ ] 보조 질문이 주요 질문을 지지함
- [ ] 모든 보조 질문이 구별됨 (중복 없음)

## 범위 정의
- [ ] 포함 범위 항목이 명확히 나열됨
- [ ] 제외 범위 항목이 명시적으로 기술됨
- [ ] 범위가 가용 시간/자원에 현실적임

## 성공 기준
- [ ] 기준이 측정 가능함
- [ ] 기준이 달성 가능함
- [ ] 기준이 모든 보조 질문을 다룸

## 제약조건
- [ ] 시간 제약이 문서화됨
- [ ] 자원 제한이 기록됨
- [ ] 접근 제한이 식별됨
```

### 방법론 단계용:

```markdown
# 품질 체크리스트: 방법론

## 연구 설계
- [ ] 접근법이 질문 유형과 일치함
- [ ] 데이터 출처가 질문에 적절함
- [ ] 분석 방법이 정의됨

## 검색 전략
- [ ] 각 질문에 키워드가 식별됨
- [ ] 여러 출처 카테고리가 계획됨
- [ ] 품질 기준이 수립됨

## 품질 기준
- [ ] 최소 출처 수가 정의됨
- [ ] 품질 등급 시스템이 지정됨
- [ ] 증거 삼각검증이 계획됨
```

### 검색/문헌 단계용:

```markdown
# 품질 체크리스트: 출처

## 출처 커버리지
- [ ] 질문당 최소 출처 수 충족
- [ ] 여러 출처 카테고리 사용됨
- [ ] 최근과 기초 출처가 포함됨

## 출처 품질
- [ ] 50% 이상이 A 또는 B 등급
- [ ] 단일 출처에 과도한 의존 없음
- [ ] 다양한 관점 포함됨

## 문서화
- [ ] 모든 출처가 참고문헌에 있음
- [ ] 품질 등급이 할당됨
- [ ] 질문과의 관련성이 기록됨
```

### 수집 단계용:

```markdown
# 품질 체크리스트: 증거

## 커버리지
- [ ] 모든 질문에 대해 증거 수집됨
- [ ] 질문당 3개 이상의 증거
- [ ] 핵심 주장에 A등급 증거 있음

## 품질
- [ ] 모든 증거가 출처로 추적 가능
- [ ] 인용문의 맥락이 보존됨
- [ ] 반박 증거가 포함됨

## 정리
- [ ] 증거가 질문별로 정리됨
- [ ] 각 질문에 요약이 생성됨
- [ ] 커버리지 매트릭스가 완성됨
```

### 분석 단계용:

```markdown
# 품질 체크리스트: 분석

## 종합 품질
- [ ] 모든 질문이 다뤄짐
- [ ] 증거가 통합됨 (단순 나열이 아님)
- [ ] 패턴/주제가 식별됨
- [ ] 모순이 다뤄짐

## 엄밀성
- [ ] 핵심 주장에 여러 출처가 지지함
- [ ] 대안적 해석이 고려됨
- [ ] 한계가 인정됨

## 논리
- [ ] 결론이 증거에서 도출됨
- [ ] 논리적 오류 없음
- [ ] 신뢰도 수준이 적절함
```

### 발견사항 단계용:

```markdown
# 품질 체크리스트: 발견사항

## 내용 품질
- [ ] 요약이 독립적임
- [ ] 핵심 발견사항이 명확히 기술됨
- [ ] 모든 주장에 증거가 인용됨
- [ ] 신뢰도 수준이 명시됨

## 완전성
- [ ] 모든 연구 질문이 답변됨
- [ ] 방법론이 요약됨
- [ ] 한계가 문서화됨
- [ ] 참고문헌이 완성됨

## 신뢰성
- [ ] 지지되지 않은 주장 없음
- [ ] 반박 증거가 다뤄짐
- [ ] 편향이 인정됨
```

### 3. 체크리스트 평가

각 항목에 대해:
1. 관련 산출물 확인
2. 통과/실패 판단
3. 실패 시 구체적 문제 기록

### 4. 품질 보고서 생성

```markdown
# 품질 보고서: [단계]

## 요약
- **상태**: [통과 / 주의 필요 / 심각한 문제]
- **통과한 확인**: [#]/[#]
- **심각한 문제**: [#]

## 상세 결과

### 통과한 확인
- [x] [확인 항목 1]
- [x] [확인 항목 2]

### 실패한 확인
- [ ] [확인 항목 3]
  - **문제**: [구체적 문제]
  - **수정**: [권장 조치]

- [ ] [확인 항목 4]
  - **문제**: [구체적 문제]
  - **수정**: [권장 조치]

## 권장 조치
1. [우선 조치 1]
2. [우선 조치 2]

## 다음 단계
[결과에 따라 다음에 해야 할 일]
```

### 5. 종합 최종 확인

연구가 완료되면 모든 체크리스트 실행:

```markdown
# 최종 품질 평가

## 단계별 요약
| 단계 | 상태 | 통과율 | 심각한 문제 |
|------|------|--------|-------------|
| 질문 | 통과 | 100% | 0 |
| 방법론 | 통과 | 90% | 0 |
| 출처 | 주의 | 80% | 0 |
| 증거 | 통과 | 95% | 0 |
| 분석 | 통과 | 100% | 0 |
| 발견사항 | 주의 | 85% | 1 |

## 전체 상태
**[완료 준비됨 / 수정 필요]**

## 미해결 문제
[전 단계에 걸친 모든 미해결 문제 목록]

## 최종 권고사항
[연구 완료 전 해야 할 일]
```

## 출력

보고:
- 감지된 현재 단계
- 체크리스트 결과
- 발견된 문제
- 권장 조치
- 전체 품질 상태

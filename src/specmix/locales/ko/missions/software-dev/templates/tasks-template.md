# Tasks: [FEATURE NAME]

**입력**: `/specs/[###-feature-name]/`의 설계 문서
**전제조건**: plan.md (필수), spec.md (사용자 스토리용 필수)

## Vertical Slice 접근법

각 Work Package(WP)는 **완전한 Vertical Slice**입니다:
- 필요한 모든 레이어 포함 (UI → API → DB)
- 독립적으로 테스트 및 배포 가능
- 사용자에게 보이는 가치 전달
- 별도로 리뷰하고 머지 가능

```
┌─────────────────────────────────────────────────────────────┐
│                    기능: 사용자 인증                          │
├──────────────┬──────────────┬──────────────┬───────────────┤
│    WP01      │    WP02      │    WP03      │    WP04       │
│   로그인     │  회원가입    │  비밀번호    │   프로필      │
│              │              │   재설정     │               │
├──────────────┼──────────────┼──────────────┼───────────────┤
│  UI 레이어   │  UI 레이어   │  UI 레이어   │  UI 레이어    │
│  API 레이어  │  API 레이어  │  API 레이어  │  API 레이어   │
│  DB 레이어   │  DB 레이어   │  DB 레이어   │  DB 레이어    │
│  테스트      │  테스트      │  테스트      │  테스트       │
├──────────────┼──────────────┼──────────────┼───────────────┤
│ ✓ 배포 가능 │ ✓ 배포 가능 │ ✓ 배포 가능 │ ✓ 배포 가능   │
└──────────────┴──────────────┴──────────────┴───────────────┘
```

## 작업 추적

- **tasks.md** (이 파일): 체크박스 형식의 개요
- **Work Package 파일**: `tasks/{planned,doing,for_review,done}/WPxx.md`
  - 상세 구현 노트
  - 대시보드 칸반 시각화
  - `/spec-mix.review` 연동

### WP00: 프로젝트 부트스트랩 (Foundation)

**Vertical Scope**: 프로젝트 구조 + 빌드 + 실행 가능성

| 레이어 | 산출물 |
|--------|--------|
| Config | 프로젝트 초기화, 의존성, 환경 설정 |
| Infrastructure | 빌드 스크립트, CI 설정 (필요시) |
| Verification | `npm start` / `python main.py` / 동등한 명령이 오류 없이 실행 |

**작업**:
- [ ] T001 구현 계획에 따른 프로젝트 구조 생성
- [ ] T002 프레임워크 및 의존성으로 초기화

**체크포인트**: 프로젝트 스켈레톤 실행됨 → Vertical Slice 준비 완료

## 의존성 맵

```
WP00 (Foundation)
 │
 ├──→ WP01 (로그인) ────→ WP00 이후 배포 가능
 │
 ├──→ WP02 (회원가입) ──→ WP00 이후 배포 가능 (WP01과 병렬)
 │
 ├──→ WP03 (비밀번호) ──→ WP00 이후 배포 가능 (병렬)
 │
 └──→ WP04 (프로필) ───→ WP01 의존 가능 (로그인 필요)
      │
      └──→ WP99 (폴리시) → 모든 코어 슬라이스 이후
```

## 실행 전략

### 순차 (솔로 개발자)
```
WP00 → WP01 → 배포 → WP02 → 배포 → WP03 → 배포 → WP04 → WP99
```

### 병렬 (팀)
```
WP00 (함께)
  ↓
WP01 (개발자 A) ──→ 배포
WP02 (개발자 B) ──→ 배포
WP03 (개발자 C) ──→ 배포
  ↓
WP04 (누구나) ───→ 배포
  ↓
WP99 (함께)
```

### MVP 우선
```
WP00 → WP01 → WP02 → 중단 & 검증 → MVP 배포
       (이후 WP03, WP04 계속...)
```

## Work Package 파일 형식

각 WP는 `tasks/planned/WPxx.md`에 상세 파일을 가질 수 있음:

```markdown
---
id: WP01
title: 사용자 로그인
status: planned
priority: P1
vertical_scope: login-flow
layers: [ui, api, db, test]
deployable: true
dependencies: [WP00]
---

# WP01: 사용자 로그인

## Vertical Scope
UI에서 데이터베이스까지 완전한 로그인 플로우

## Layers
- **UI**: 로그인 폼 컴포넌트
- **API**: POST /api/auth/login
- **DB**: users 테이블, sessions 테이블
- **Test**: 로그인 통합 테스트

## 작업
- [ ] users 마이그레이션 생성
- [ ] 로그인 엔드포인트 구현
- [ ] 로그인 폼 생성
- [ ] 통합 테스트 추가

## 인수 조건
- [ ] 사용자가 이메일/비밀번호로 로그인 가능
- [ ] 잘못된 자격 증명 시 오류 표시
- [ ] 새로고침 후에도 세션 유지
```

## 참고 사항

- **Vertical Slice**: 각 WP는 기능에 필요한 모든 레이어 포함
- **독립 배포**: 완료된 각 WP는 다른 것 없이 배포 가능
- **레이어 라벨**: [DB], [API], [UI], [Test]로 레이어 표시
- **의존성**: WP00만 다른 것을 차단; 나머지는 병렬 가능
- **Walkthrough**: 각 WP 완료 시 작업 메모리용 walkthrough 생성

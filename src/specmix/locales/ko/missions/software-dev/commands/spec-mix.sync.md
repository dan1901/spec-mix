## 사용자 입력

```text
$ARGUMENTS
```

진행하기 전에 사용자 입력을 **반드시** 고려해야 합니다 (비어있지 않은 경우).

## 목적

이 명령어는 기존 산출물을 모두 읽어 프로젝트의 현재 상태를 빠르게 파악하도록 돕습니다. 다음 상황에서 사용하세요:

- 이 프로젝트를 작업하던 다른 AI 에이전트로부터 전환할 때
- 휴식 후 새 세션을 시작할 때
- 기존 프로젝트에 온보딩할 때
- 무엇이 완료되었고 다음은 무엇인지 파악해야 할 때

## 실행 흐름

### 1. 프로젝트 컨텍스트 파악

먼저 기본 프로젝트 정보를 수집합니다:

```bash
# 프로젝트 설정 확인
cat .spec-mix/config.json
```

다음 사항을 확인하세요:
- 현재 모드 (normal/pro)
- 언어 설정
- 활성 미션
- 기본 AI 에이전트

### 2. 활성 기능 찾기

현재 작업 중인 기능을 확인합니다:

```bash
# 현재 브랜치 확인
git branch --show-current

# 모든 기능 스펙 목록
ls specs/
```

기능 브랜치(예: `003-user-auth`)에 있다면 그것이 활성 기능입니다.
main에 있다면 가장 최근에 수정된 spec 디렉토리를 찾으세요.

### 3. 핵심 산출물 읽기

활성 기능에 대해 다음 파일들을 순서대로 읽습니다:

#### a. 사양서 (What)
```bash
cat specs/{feature}/spec.md
```
- 기능 요구사항 이해
- 사용자 스토리 및 수락 기준 확인
- 상태 필드 확인

#### b. 계획 (How)
```bash
cat specs/{feature}/plan.md
```
- 기술적 접근 방식 이해
- 아키텍처 결정 사항 확인
- 구현 단계 검토

#### c. 작업 (Progress)
```bash
cat specs/{feature}/tasks.md
```
- 완료된 작업 `[x]` 확인
- 진행 중인 작업 확인
- 대기 중인 작업 확인

#### d. 작업 레인 (상세 상태)
```bash
ls specs/{feature}/tasks/planned/
ls specs/{feature}/tasks/doing/
ls specs/{feature}/tasks/for_review/
ls specs/{feature}/tasks/done/
```
- 각 레인의 작업 수 확인
- 차단된 작업 확인

### 4. 최근 워크스루 읽기

워크스루에는 완료된 작업에 대한 귀중한 컨텍스트가 있습니다:

```bash
ls -lt specs/{feature}/walkthroughs/
```

가장 최근 워크스루 파일을 읽어 다음을 파악하세요:
- 구현된 내용
- 주요 결정 사항
- 발생한 문제

### 5. 프로젝트 노트 확인

```bash
cat .spec-mix/notes.md 2>/dev/null || echo "노트 없음"
```

노트에는 다음이 포함될 수 있습니다:
- 이전 세션의 중요한 알림
- 경고 또는 주의사항
- 다른 에이전트의 인수인계 메시지

### 6. 헌장 확인

```bash
cat .spec-mix/memory/constitution.md 2>/dev/null
```

프로젝트 원칙과 지침을 이해합니다.

## 출력: 컨텍스트 요약

모든 산출물을 읽은 후 요약을 제시합니다:

```markdown
# 🔄 프로젝트 동기화 완료

## 프로젝트 개요
- **프로젝트**: {project-name}
- **모드**: {normal/pro}
- **언어**: {en/ko}
- **미션**: {software-dev/research}

## 활성 기능: {feature-number}-{feature-name}

### 현재 상태
- **사양서**: ✅ 완료
- **계획**: ✅ 완료
- **작업**: 🔄 진행 중 (6/10 완료)

### 작업 진행 상황
| 레인 | 개수 |
|------|------|
| ✅ 완료 | 4 |
| 🔍 검토 대기 | 2 |
| 🔨 진행 중 | 1 |
| 📋 계획됨 | 3 |

### 최근 활동
- 2단계 완료 (`phase-2-walkthrough.md` 참조)
- 현재 작업 중: WP05 - API 통합
- 마지막 업데이트: {timestamp}

### 이전 세션의 노트
{노트 또는 인수인계 메시지}

## 제안하는 다음 단계
1. `doing/`에서 WP05 (API 통합) 계속
2. `for_review/`의 완료된 작업 검토
3. {컨텍스트 기반 기타 제안}

## 빠른 명령어
- `/spec-mix.implement` - 구현 계속
- `/spec-mix.review` - 완료된 작업 검토
- `spec-mix note "메시지"` - 다음 세션을 위한 노트 추가
```

## 모범 사례

1. **에이전트 전환 또는 새 세션 시작 시 항상 먼저 동기화**
2. **워크스루 읽기** - 가장 최근 컨텍스트가 포함됨
3. **노트 확인** - 이전 에이전트가 중요한 메시지를 남겼을 수 있음
4. **이해 확인** - 불분명한 점이 있으면 사용자에게 확인 요청

## 관련 명령어

- `/spec-mix.implement` - 동기화 후 구현 계속
- `/spec-mix.review` - 완료된 작업 검토
- `spec-mix note` - 인수인계를 위한 노트 추가/읽기

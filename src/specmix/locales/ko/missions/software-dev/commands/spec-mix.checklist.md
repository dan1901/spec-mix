## 사용자 입력

```text
$ARGUMENTS

```text
진행하기 전에 사용자 입력을 **반드시** 고려해야 합니다(비어있지 않은 경우).

## 개요

이 명령은 "영어를 위한 단위 테스트"처럼 사양 및 계획 문서의 품질을 검증하기 위한 체크리스트를 생성합니다.

## 체크리스트 유형

1. **사양 품질 체크리스트**:
   - 요구사항 완전성
   - 명확성 및 모호성 없음
   - 테스트 가능성
   - 기술 중립성

2. **계획 품질 체크리스트**:
   - 아키텍처 일관성
   - 기술 스택 정당성
   - 확장성 고려사항
   - 보안 및 성능 요구사항

3. **작업 품질 체크리스트**:
   - 작업 크기 적정성
   - 명확한 수락 기준
   - 종속성 추적
   - 테스트 전략

## 실행 흐름

1. **문서 분석**:
   - 대상 아티팩트 읽기
   - 품질 기준 식별
   - 검증 포인트 추출

2. **체크리스트 생성**:
   - 문서 유형에 맞는 체크리스트 구조 사용
   - 구체적이고 실행 가능한 항목 생성
   - 우선순위 및 카테고리 할당

3. **초기 검증**:
   - 각 체크리스트 항목에 대해 문서 평가
   - 통과/실패 상태 표시
   - 문제 영역 강조

4. **출력**:
   - `checklists/` 디렉토리에 체크리스트 저장
   - 검증 결과 요약 제공
   - 개선 권장사항 제시

## 사용 시기

- `/spec-mix.plan` 후: 사양 완전성 검증

- `/spec-mix.tasks` 후: 계획 및 작업 품질 검증

- 반복 개선 중: 진행 상황 추적

## 출력 형식

체크리스트는 마크다운 형식으로 저장되며 다음을 포함합니다:
- [ ] 미완료 항목

- [x] 완료된 항목

- 각 항목에 대한 세부 설명

- 참조 섹션 링크

---
description: 6-Pager 문서를 검토하고 피드백을 정리합니다.
---

## 사용자 입력

```text
{ARGS}
```

위 입력이 비어있지 않다면 반드시 고려해야 합니다.

## 개요

이 명령어는 6-Pager 문서를 체계적으로 검토하고, 이해관계자 피드백을 수집/정리하는 것을 돕습니다.

---

## 작업 절차

### 1단계: 문서 로드

```bash
# 6-Pager 문서 읽기
cat specs/strategy/6pager.md

# 분석 문서 확인
ls -la specs/strategy/*.md
```

### 2단계: 검토 유형 선택

사용자 입력에 따라 검토 유형을 결정합니다:

| 입력 | 검토 유형 |
|------|-----------|
| `self` | 자체 품질 검토 |
| `peer` | 피어 리뷰 지원 |
| `stakeholder` | 이해관계자 리뷰 지원 |
| (빈 입력) | 종합 검토 |

---

## 검토 유형별 절차

### A. 자체 품질 검토 (`self`)

문서의 완성도와 품질을 점검합니다:

```markdown
## 자체 품질 검토 체크리스트

### 1. 구조 완성도

| 섹션 | 완성 여부 | 비고 |
|------|-----------|------|
| 1. 한 줄 요약 | ✅/❌ | |
| 2. 배경과 목적 | ✅/❌ | |
| 3. 목표 & KPI | ✅/❌ | |
| 4. 시장 크기 | ✅/❌ | |
| 5. 고객 이해 | ✅/❌ | |
| 6. 경쟁사 비교 | ✅/❌ | |
| 7. 해결책 & USP | ✅/❌ | |
| 8. User Stories | ✅/❌ | |
| 9. 비즈니스 모델 | ✅/❌ | |
| 10. GTM 전략 | ✅/❌ | |
| 11. 제품 원칙 | ✅/❌ | |
| 12. 마일스톤 | ✅/❌ | |
| 13. 리스크 | ✅/❌ | |
| 14. 오픈 이슈 | ✅/❌ | |

### 2. 품질 기준 점검

#### 명확성 (Clarity)
- [ ] 전문 용어가 적절히 정의되어 있는가?
- [ ] 문장이 명확하고 이해하기 쉬운가?
- [ ] 논리적 흐름이 자연스러운가?

#### 구체성 (Specificity)
- [ ] 추상적 표현 대신 구체적 수치가 있는가?
- [ ] 예시가 충분히 제공되어 있는가?
- [ ] 타겟 고객이 명확하게 정의되어 있는가?

#### 데이터 품질 (Data Quality)
- [ ] 모든 수치에 출처가 명시되어 있는가?
- [ ] 가정과 사실이 명확히 구분되어 있는가?
- [ ] 최신 데이터인가?

#### 실행 가능성 (Actionability)
- [ ] 다음 단계가 명확한가?
- [ ] 담당자와 기한이 지정되어 있는가?
- [ ] 리스크에 대한 대응책이 있는가?

### 3. 개선 필요 항목

| 우선순위 | 항목 | 현재 상태 | 개선 방향 |
|----------|------|-----------|-----------|
| High | | | |
| Medium | | | |
| Low | | | |
```

### B. 피어 리뷰 지원 (`peer`)

동료 PM의 리뷰를 위한 체크리스트와 질문을 생성합니다:

```markdown
## 피어 리뷰 가이드

### 리뷰어를 위한 질문

#### 전략 정렬
1. 이 제품이 왜 지금 필요한지 충분히 설득력 있는가?
2. 목표와 KPI가 비전에 부합하는가?
3. 타겟 고객 선정이 적절한가?

#### 시장 분석
1. TAM/SAM/SOM 산출 근거가 타당한가?
2. 경쟁 환경 분석이 충분한가?
3. 차별화 포인트가 지속 가능한가?

#### 실행 계획
1. 마일스톤이 현실적인가?
2. 리스크가 충분히 고려되었는가?
3. GTM 전략이 타겟 고객에게 적합한가?

#### 비즈니스 모델
1. Unit Economics 가정이 합리적인가?
2. 가격 전략이 시장과 고객에게 적합한가?
3. 수익 모델의 확장성이 있는가?

### 피드백 작성 양식

```
### 피드백: [섹션명]

**현재 상태**:
[현재 문서의 해당 부분 요약]

**우려 사항**:
[무엇이 문제인지]

**제안**:
[어떻게 개선해야 하는지]

**참고 자료**:
[관련 데이터나 사례]
```
```

### C. 이해관계자 리뷰 지원 (`stakeholder`)

이해관계자별 맞춤 리뷰 자료를 준비합니다:

```markdown
## 이해관계자 리뷰 준비

### 이해관계자별 초점

| 이해관계자 | 주요 관심사 | 검토 섹션 |
|------------|-------------|-----------|
| CEO/경영진 | 전략 정렬, ROI | 1, 2, 3, 12 |
| CFO/재무 | Unit Economics, 예산 | 4, 9, 12 |
| CTO/개발 | 기술 실현 가능성 | 7, 8, 12, 13 |
| 마케팅 | GTM, 포지셔닝 | 5, 6, 10 |
| 영업 | 가격, 경쟁 | 6, 7, 9 |
| 법무 | 규제, 리스크 | 13, 14 |

### 경영진 브리핑 요약

```
## Executive Summary

**한 줄 요약**:
[6-Pager의 한 줄 요약]

**핵심 기회**:
- [기회 1]
- [기회 2]

**필요 투자**:
- [자원 1]
- [자원 2]

**주요 리스크**:
- [리스크 1]
- [리스크 2]

**요청 사항**:
- [승인/의사결정 필요 사항]
```

### 리뷰 미팅 안건

1. **소개** (5분)
   - 제품 개요
   - 리뷰 목적

2. **핵심 내용 발표** (15분)
   - 문제 정의
   - 솔루션 개요
   - 시장 기회

3. **Q&A 및 피드백** (20분)
   - 부서별 질문
   - 우려 사항 논의

4. **다음 단계** (5분)
   - 피드백 반영 계획
   - 승인 타임라인
```

## 종합 검토 결과

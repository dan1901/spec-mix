## 사용자 입력

```text
{ARGS}
```

위 입력이 비어있지 않다면 반드시 고려해야 합니다.

## 개요

이 명령어는 Product Owner 관점의 6-Pager 전략 문서를 작성합니다. Amazon 스타일의 6-Pager 형식을 따르며, 다음 14개 섹션으로 구성됩니다:

1. 한 줄 요약
2. 배경과 목적
3. 목표 & KPI
4. 시장 크기 (TAM/SAM/SOM)
5. 고객 이해 (페르소나)
6. 경쟁사 비교
7. 해결책 & USP
8. User Stories
9. 비즈니스 모델
10. GTM 전략
11. 제품 원칙
12. 마일스톤
13. 리스크
14. 오픈 이슈

---

## 작업 절차

### 1단계: 제품 아이디어 파악

사용자 입력이 없거나 불충분한 경우, 다음을 질문합니다:

```
6-Pager 전략 문서를 작성하겠습니다.

다음 정보를 알려주세요:

1. **제품명**: 제품/서비스의 이름
2. **한 줄 설명**: 무엇을 하는 제품인가요?
3. **타겟 고객**: 누구를 위한 제품인가요?
4. **해결하려는 문제**: 어떤 문제를 해결하나요?
5. **기존에 있는 자료**: (있다면) 기획서, 리서치 자료, 아이디어 메모 등
```

### 2단계: 기존 자료 분석

프로젝트에 이미 존재하는 자료가 있는지 확인합니다:

```bash
# 기존 전략 문서 확인
ls -la specs/*/6pager.md 2>/dev/null
ls -la docs/*.md 2>/dev/null

# 리서치 자료 확인
ls -la specs/*/market-research.md 2>/dev/null
ls -la specs/*/competitor-analysis.md 2>/dev/null
```

기존 자료가 있다면 읽어서 내용을 반영합니다.

### 3단계: 정보 수집 대화

각 섹션별로 필요한 정보를 대화를 통해 수집합니다. 한 번에 너무 많은 질문을 하지 말고, 섹션별로 나누어 진행합니다.

#### 3.1 배경과 목적 질문

```
## 배경과 목적

1. **왜 지금** 이 제품을 만들어야 하나요?
   - 시장에 어떤 변화가 있나요?
   - 기술적으로 가능해진 것이 있나요?
   - 고객 행동에 변화가 있나요?

2. **어떤 문제**를 해결하려고 하나요?
   - 고객이 현재 겪고 있는 불편함은?
   - 기존 솔루션의 한계는?
```

#### 3.2 목표 & KPI 질문

```
## 목표 & KPI

1. **3-5년 후 비전**은 무엇인가요?
   - 이 제품이 성공하면 어떤 모습일까요?

2. **측정 가능한 목표**는 무엇인가요?
   - 1년 후 달성하고 싶은 수치 목표가 있나요?
   - (예: MAU 10만, 전환율 5%, NPS 50 이상)
```

#### 3.3 시장 크기 질문

```
## 시장 크기

1. **어떤 시장**을 타겟으로 하나요?
   - 시장의 범위를 어떻게 정의하시나요?

2. **시장 규모**에 대해 알고 있는 정보가 있나요?
   - 전체 시장 규모 (TAM)
   - 접근 가능한 시장 (SAM)
   - 획득 목표 시장 (SOM)

3. **시장 성장률**은 어떤가요?
```

#### 3.4 고객 이해 질문

```
## 고객 이해

1. **핵심 타겟 고객**은 누구인가요?
   - 나이, 직업, 특성
   - 구체적인 페르소나가 있나요?

2. **고객의 Pain Point**는 무엇인가요?
   - 현재 어떤 방식으로 문제를 해결하고 있나요?
   - 그 방식의 불편한 점은?

3. **고객이 원하는 것**은 무엇인가요? (JTBD)
   - 기능적으로 해결하고 싶은 것
   - 감정적으로 원하는 것
   - 사회적으로 보여주고 싶은 것
```

#### 3.5 경쟁사 질문

```
## 경쟁사 분석

1. **직접 경쟁사**는 누구인가요?
   - 같은 문제를 해결하는 제품/서비스

2. **간접 경쟁사 또는 대체재**는?
   - 다른 방식으로 같은 니즈를 충족시키는 것

3. **경쟁사 대비 우리의 강점/약점**은?
```

#### 3.6 솔루션 & USP 질문

```
## 솔루션 & USP

1. **핵심 기능** 3-5가지는 무엇인가요?
   - 각 기능이 고객에게 주는 가치는?

2. **차별화 포인트** (USP)는 무엇인가요?
   - 경쟁사가 따라하기 어려운 우리만의 강점

3. **MVP 범위**는 어디까지인가요?
   - 첫 버전에 반드시 들어가야 할 기능
```

#### 3.7 비즈니스 모델 질문

```
## 비즈니스 모델

1. **수익 모델**은 무엇인가요?
   - 구독? 광고? 거래 수수료? 라이선스?

2. **가격 전략**은?
   - Free / Pro / Enterprise 등 플랜 구성
   - 각 플랜의 가격대

3. **Unit Economics 목표**는?
   - CAC, LTV, LTV:CAC ratio
```

#### 3.8 GTM 전략 질문

```
## GTM 전략

1. **런칭 전략**은?
   - Private Beta → Public Beta → GA 계획

2. **초기 고객 확보 전략**은?
   - 어떤 채널을 통해 첫 고객을 모을 건가요?

3. **마케팅 채널** 우선순위는?
```

#### 3.9 마일스톤 질문

```
## 마일스톤

1. **주요 마일스톤**은 무엇인가요?
   - 각 마일스톤의 목표와 기한

2. **Phase별 계획**이 있나요?
```

#### 3.10 리스크 질문

```
## 리스크

1. **가장 큰 리스크**는 무엇인가요?
   - 시장, 기술, 경쟁, 운영, 규제 측면에서

2. **각 리스크에 대한 대응 전략**은?
```

### 4단계: 6-Pager 문서 생성

수집한 정보를 바탕으로 6-Pager 문서를 생성합니다.

1. **템플릿 로드**:
   ```
   .spec-mix/active-mission/templates/6pager-template.md
   ```

2. **디렉토리 생성**:
   ```bash
   mkdir -p specs/strategy
   ```

3. **문서 작성**: 템플릿의 각 섹션을 채웁니다.
   - 확인되지 않은 정보는 `[TBD]` 또는 `[검증 필요]`로 표시
   - 가정(Assumption)은 명확히 표시
   - 수치 데이터는 출처 명시

4. **파일 저장**:
   ```
   specs/strategy/6pager.md
   ```

### 5단계: 검증 필요 항목 정리

작성 후, 다음을 정리하여 사용자에게 제시합니다:

```markdown
## 작성 완료

`specs/strategy/6pager.md` 파일이 생성되었습니다.

### 검증이 필요한 항목

다음 항목들은 추가 리서치나 검증이 필요합니다:

| 섹션 | 항목 | 검증 방법 |
|------|------|-----------|
| 시장 크기 | TAM/SAM/SOM 수치 | 시장 리서치 리포트 확인 |
| 경쟁사 | 시장 점유율 | 경쟁사 분석 |
| Unit Economics | CAC/LTV 예측 | 유사 서비스 벤치마크 |

### 다음 단계

1. `/spec-mix.analyze` - 시장/경쟁사 심층 분석
2. `/spec-mix.refine` - 피드백 반영하여 문서 개선
3. `/spec-mix.review` - 이해관계자 리뷰
```

## 작성 가이드라인

### 좋은 6-Pager의 특징

1. **구체적이고 측정 가능**: 추상적인 표현 대신 구체적인 수치와 예시
2. **데이터 기반**: 가정과 사실을 구분, 출처 명시
3. **고객 중심**: 기술이 아닌 고객 가치 관점에서 서술
4. **실행 가능**: 다음 단계가 명확한 액션 아이템
5. **간결하고 명확**: 불필요한 내용 배제

### 피해야 할 것

- 근거 없는 낙관적 예측
- 검증되지 않은 가정을 사실처럼 서술
- 기술 중심의 서술 (기술은 수단일 뿐)
- 경쟁사 과소평가
- 리스크 회피나 무시

### 정보가 부족할 때

- `[TBD]`로 표시하고 검증 필요 항목에 추가
- 가정을 세우되 명확히 `[가정]`으로 표시
- "~로 추정됨", "~로 예상됨" 등 불확실성 표현
- 검증 방법과 담당자 명시

## 예시: 한 줄 요약 작성

**나쁜 예**:
> 혁신적인 AI 기반 플랫폼으로 업계를 선도합니다.

**좋은 예**:
> 30-40대 맞벌이 부부를 위한 AI 가계부 앱으로, 은행 연동을 통해 자동으로 지출을 분류하고 월 저축 목표 달성을 돕습니다.

**좋은 예의 특징**:
- 타겟 고객이 구체적 (30-40대 맞벌이 부부)
- 무엇을 하는지 명확 (AI 가계부 앱)
- 핵심 가치가 분명 (자동 분류, 저축 목표)

# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Configuration

This project uses Spec-Driven Development (SDD) with the spec-mix toolkit.

## Walkthrough Memory Loading

**IMPORTANT**: When starting a new session, check for and load any existing walkthrough documents:

```bash
# Check if walkthrough exists for current feature
if [ -f "specs/*/walkthrough.md" ]; then
    echo "Loading previous implementation context..."
fi
```

### Walkthrough Priority

1. **First action when starting**: Look for `specs/*/walkthrough.md` files
2. **Load into context**: Read and remember the implementation decisions and work completed
3. **Use as reference**: Consider previous architectural decisions and patterns when continuing work

### Walkthrough Structure

Walkthroughs contain:

- Summary of completed work
- Files modified and rationale
- Architecture decisions made
- Testing status
- Known issues and TODOs
- Commit history

## SDD Workflow

The project follows this structured workflow:

1. `/spec-mix.constitution` - Establish project principles (optional but recommended)
2. `/spec-mix.specify` - Create feature specification
3. `/spec-mix.clarify` - Clarify ambiguities (optional)
4. `/spec-mix.plan` - Create technical plan
5. `/spec-mix.tasks` - Generate task breakdown
6. `/spec-mix.analyze` - Cross-artifact analysis (optional)
7. `/spec-mix.implement` - Execute implementation (auto-generates walkthrough)
8. `/spec-mix.review` - Review completed work
9. `/spec-mix.accept` - Final acceptance check
10. `/spec-mix.merge` - Merge to main branch

## Key Commands

- `specify init` - Initialize SDD in a project
- `specify check` - Validate environment
- `specify lang` - Manage language settings
- `specify mission` - Switch development missions
- `specify dashboard` - Launch web dashboard

## Work Session Best Practices

1. **Start of session**: Check for existing walkthroughs
2. **During implementation**: Follow lane workflow (planned → doing → for_review → done)
3. **End of session**: Walkthrough is automatically generated by `/spec-mix.implement`
4. **Context preservation**: Walkthroughs serve as session memory between AI interactions

## File Locations

- Specifications: `specs/{feature-number}-{feature-name}/`
- Walkthroughs: `specs/{feature}/walkthrough.md`
- Tasks: `specs/{feature}/tasks/{lane}/*.md`
- Scripts: `.spec-mix/scripts/`
- Commands: `.claude/commands/`

## Important Notes

- Always check for existing walkthroughs before starting work
- Commit messages should include Work Package IDs: `[WP01]`
- Follow the lane workflow strictly
- Walkthroughs are automatically generated - do not create manually

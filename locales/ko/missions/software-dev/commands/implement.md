## 사용자 입력

```text
$ARGUMENTS
```

진행하기 전에 사용자 입력을 **반드시** 고려해야 합니다(비어있지 않은 경우).

## 개요

이 명령은 `/speckit.tasks`로 생성된 작업 분석을 읽고 각 작업을 순서대로 실행하여 기능을 구현합니다.

## 실행 흐름

1. **전제조건 확인**:
   - `tasks.md`가 존재하고 완료됨
   - 모든 필수 설계 아티팩트(spec, plan, research, data-model)가 존재함

2. **작업 파싱**:
   - `tasks.md`에서 모든 작업 읽기
   - 종속성 그래프 구축
   - 실행 순서 결정

3. **작업 실행**:
   - 각 작업을 순서대로 처리
   - 병렬 작업([P] 표시)은 동시 실행 가능
   - 각 작업 후 진행 상황 보고

4. **검증 및 테스트**:
   - 각 체크포인트에서 기능 검증
   - 테스트 실행 (정의된 경우)
   - 오류 발생 시 보고 및 중지

## 주요 규칙

- 한 번에 하나의 작업만 수정
- 각 작업 완료 후 진행 상황 보고
- 오류 발생 시 명확한 오류 메시지와 함께 중지
- 모든 생성된 코드는 헌장 원칙을 준수해야 함

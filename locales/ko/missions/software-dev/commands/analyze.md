## 사용자 입력

```text
$ARGUMENTS
```

진행하기 전에 사용자 입력을 **반드시** 고려해야 합니다(비어있지 않은 경우).

## 개요

이 명령은 `/speckit.tasks` 후 `/speckit.implement` 전에 실행되어 모든 아티팩트 간의 일관성과 완전성을 검증합니다.

## 분석 영역

1. **사양-계획 정렬**:
   - 모든 기능 요구사항이 계획에 다루어졌는지 확인
   - 계획의 모든 기술 결정이 사양 제약조건을 위반하지 않는지 확인
   - 성공 기준이 구현 가능한지 확인

2. **계획-작업 정렬**:
   - 모든 계획된 구성요소가 작업에 포함되었는지 확인
   - 작업 종속성이 아키텍처와 일치하는지 확인
   - 작업 범위가 계획 범위와 일치하는지 확인

3. **사양-작업 추적성**:
   - 각 사용자 스토리가 작업에 매핑되었는지 확인
   - 각 기능 요구사항이 하나 이상의 작업으로 다루어졌는지 확인
   - 작업에 사양에 없는 구현이 포함되지 않았는지 확인

4. **헌장 준수**:
   - 계획 및 작업이 프로젝트 원칙을 준수하는지 확인
   - 복잡성 게이트 위반 확인
   - 필수 품질 게이트 포함 확인

## 출력

분석 보고서는 다음을 포함합니다:

- **✅ 통과**: 정렬되고 일관된 영역
- **⚠️ 경고**: 잠재적 문제 또는 개선 영역
- **❌ 오류**: 구현 전 해결해야 하는 문제

## 권장 사항

- 오류가 있는 경우: 구현 전 해당 아티팩트 수정
- 경고가 있는 경우: 검토 및 필요시 수정
- 모든 것이 통과한 경우: `/speckit.implement` 진행 가능
